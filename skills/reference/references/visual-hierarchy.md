# Visual Hierarchy

Distilled from *Design for Developers* (basics; full chapter is future scope).

## What & why

Visual hierarchy ranks elements in the order you want users to see them, guiding them to desired
actions. Done right, it **reduces the effort** to find things — removing friction and improving
usability. Users **scan, not read**, so build the hierarchy deliberately.

## Scan patterns

- **F-pattern** — for **information-heavy** pages. Eyes go left→right, top→bottom. Place key items along
  the F's strokes.
- **Z-pattern** — for **light/sparse** pages. Logo (top-left) → nav (top-right) → diagonal to first key
  content → across the bottom. Put the logo top-left (seen first); primary CTA on the Z's path.

## The levers (how to alter hierarchy)

Size · Text · Color · Contrast · Proximity · Alignment · Consistency · Negative space ·
Visual harmony · (plus buttons & inputs as focal elements).

- **Size & text** — bigger = scanned first, smaller = last. Increase the most important text's font
  size so it's seen first; titles first, lead paragraphs second, minor sections last. Size can
  **override reading order** (a large name reads before a small "years of experience" placed above it).
- **Color & contrast** — warm/saturated/high-contrast elements advance; muted/low-contrast recede.
  Make the primary CTA pop. View in greyscale to test that hierarchy survives on contrast alone.
- **Proximity** — related items sit close (read as a group); unrelated items get space.
- **Alignment** — consistent alignment creates order and an invisible grid the eye follows.
- **Consistency** — repeated patterns (spacing, sizes, colors) let users predict and scan faster.
- **Negative space** — generous space around an element elevates its importance and gives clear focal
  points. Don't cram elements together — clutter destroys focal points and few users will parse it.
- **Visual harmony** — colors, type, and spacing all drawn from the same systems so the page feels unified.

## DO / DON'T (summary)

- **Do** make the primary CTA stand out; keep clear hierarchy between text and images; leave
  appropriate space between elements.
- **Don't** cluster elements with little/no space (no clear focal points); don't clutter the UI.
